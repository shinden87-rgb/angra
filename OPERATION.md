# Angra - Research Hub Operation Manual

本番稼働おめでとうございます！
このドキュメントは、今後の「論文データの定期更新」をスムーズに行うための運用マニュアルです。

## 🔄 定期更新フロー

このプロジェクトは **「JSONファイルを更新してPushするだけ」** でサイトが更新される構成になっています。

### 手順

1. **AIエージェントに依頼する**
   作成したGem（Angra Dev Buddy）に以下のプロンプトを投げます。
   > 「最新の健康・トレーニングに関する論文をリサーチして、news-feed.jsonに追加してください。」

2. **内容の確認**
   - AIが提示した論文のURLが正しいか（リンク切れでないか）確認します。
   - `npm run dev` でローカルサーバーを立ち上げ、`http://localhost:4000/research-news` で表示を確認します。
   - バッジの色（Meta-Analysisなら紫など）が正しいか見ます。

3. **反映（Deploy）**
   問題なければ、以下のコマンドを実行してGitHubにプッシュします。

   ```bash
   git add .
   git commit -m "update: Add new research papers (YYYY-MM-DD)"
   git push
   ```

4. **完了**
   - Vercelが自動的に検知し、数分以内に本番サイトが更新されます。

## 🛠 将来的な改善案 (To-Do)

データが増えてきたら、以下の改修を検討してください。

- **ページネーション (Pagination)**: 現在は全件表示しているため、記事が50件を超えるとページが重くなります。「もっと見る」ボタンやページ送りの実装が必要です。
- **フィルタリング機能**: `study_type` や `category` で絞り込む機能があると便利です。
- **アーカイブ化**: `news-feed-2025.json` のように年ごとにファイルを分ける運用も検討可能です。
